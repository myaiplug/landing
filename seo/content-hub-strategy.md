## Internal Linking Structure

### Link Graph Architecture

```
                    ┌─────────────────────┐
                    │   HOMEPAGE (/)      │
                    │   Authority: 1.0    │
                    └─────────┬───────────┘
                              │
            ┌─────────────────┼──────────────────┐
            ▼                 ▼                   ▼
    ┌──────────────┐ ┌───────────────┐ ┌────────────────┐
    │  /marketplace│ │   /tools      │ │   /blog        │
    │  Authority   │ │  Authority    │ │  Authority     │
    │  0.9         │ │  0.9          │ │  0.8           │
    └──────┬───────┘ └──────┬────────┘ └───────┬────────┘
           │                │                   │
    ┌──────┴──────┐  ┌──────┴────────┐  ┌──────┴──────────┐
    │Product Pages│  │Tool Pages     │  │Pillar Page      │
    │20 pages     │  │3 pages        │  │1 page           │
    │Authority 0.9│  │Authority 0.8  │  │Authority 0.8    │
    └─────────────┘  └───────────────┘  └──────┬──────────┘
                                               │
                                     ┌─────────┴─────────┐
                                     │   Cluster Content  │
                                     │   25-35 articles   │
                                     │   Authority 0.6-0.7│
                                     └───────────────────┘
```

### Linking Rules

1. **Every blog post** links to the pillar page in the first 200 words
2. **Every blog post** links to its relevant product/tool page in a CTA at the end
3. **Every product page** links to related blog posts in a "Learn More" section
4. **Every tool page** links to the pillar page and relevant blog posts
5. **Pillar page** links to every cluster article (linked list at bottom)
6. **Homepage** links to the pillar page, top 3 products, and top 3 tools
7. **Category pages** link to products and relevant blog posts

### Anchor Text Strategy
- Use exact match for high-value keywords (10% of links)
- Use partial match for semantic keywords (40% of links)
- Use branded anchors for brand authority (30% of links)
- Use generic CTAs for user experience (20% of links)

### Internal Link Density
- Pillar page: 20-30 internal links
- Blog posts: 5-8 internal links per 1000 words
- Product pages: 5-10 internal links
- Tool pages: 8-12 internal links

## Emergency Content Playbook

### When a Tool Breaks or Changes
- Publish "Troubleshooting" post within 24 hours
- Update all affected articles with new instructions
- Add banner on tool page linking to updated guide

### When a Competitor Launches
- Write comparison article within 1 week
- Highlight differentiators
- Update pillar page with new competitor entry
- Pitch "X vs. Y" articles to music blogs

### Algorithm Update
- Audit top 20 pages for quality
- Add missing schema
- Improve internal linking
- Update freshness dates

### Seasonal Opportunities
- **New Year:** "2026 Music Production Resolutions" (Jan 1)
- **Summer:** "Make Summer Hits with AI Tools" (Jun)
- **Holiday:** "AI Music Gifts for Producers" (Dec)
- **Music Festivals:** Tie content to major events (SXSW, NAMM, etc.)
